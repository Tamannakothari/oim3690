# API
API: Application Programming Interface
- It is a way for programs to talk to each other
- Can be thought of as a restaurant
    - Menu (API Documentation) - shows what you can order
    - Waiter (API) - takes your request, brings back food
    - Kitchen (Server) - prepares what you ordered

# GET vs POST
- GET = give me data (reading)
- POST = here's some data (writing)

# API Keys: Some APIs need a key. Never commit keys to GitHub
( Anyone can steal it)

# Solution: Separate Config File
1. Step 1: Create config.js with your key:
// config.js - DO NOT commit this file!
const API_KEY = 'sk-abc123...';
2. Step 2: Add it to .gitignore:
config.js
3. Step 3: Load it in your HTML before your main script:
<script src="config.js"></script>
<script src="app.js"></script>
